## 2026-05-21

### Run Ingestion page refactored to support 3 trigger APIs

Based on the Ingestion API Guide (20260520.docx), the "Run Ingestion" page (`/config`) now supports 3 trigger modes:

1. **Full Sync (default)** — `POST /collector/trigger/event` with `{ startTime, endTime }`
2. **By Table** — `POST /collector/trigger/event/table` with `{ startTime, endTime, tableName }` (retains modelId)
3. **By Record** — `POST /collector/trigger/event/record` with `{ tableName, records }` — uses JSON editor input

**Files modified:**
- `src/services/collectorService.ts` — Added `triggerEvent()`, `triggerEventByRecord()` methods; renamed old `TriggerEventRequest` to `TriggerEventByTableRequest`; added `TriggerMode` type export
- `src/components/ConfigurationForm.tsx` — Complete rewrite: Step 0 trigger mode selector (3 cards), conditional wizard (Module/Model/Table steps hidden for default mode), JSON editor for By Record, dynamic submit handler, "Run Ingestion" button, success indicator with link to Monitor
- `src/pages/Config.tsx` — Page title updated from "Test Config" to "Run Ingestion"

**TypeScript build and Vite build both pass with zero errors.**
